About Time Service & BPM Shortwave Time Station

Time service is one of the core infrastructures of modern society. Precise time synchronization is the foundation for the normal operation of satellite navigation, 5G communication, financial transactions, deep space exploration and other fields.

BPM Shortwave Time Station is China's national shortwave time service station, similar to the WWV/WWVH stations in the United States. It broadcasts precise time signals on multiple shortwave frequencies, allowing anyone with a shortwave radio to sync their clocks to national standard time for free.

Settings Guide

By adjusting the three parameters below, you can make the simulator's output match the real BPM signal as closely as possible – including UTC, time code*, UT1, and station ID.

* Author's note: To be honest, I've never actually heard the real BPM time code signal myself – or rather, it was almost always drowned out by the time code subcarrier of WWV/H. This simulator is built strictly according to the published paper (see reference [1]) – translating the encoding and broadcast rules into sound. The main goal is simple: to hear what the time code of BPM actually sounds like – that low 125 Hz hum between the second ticks – and get a feel for how it works.

Rest assured, once you set the parameters below correctly, what comes out of your browser will be identical to the real BPM broadcast – at least to the ear. If you can actually hear the time code subcarrier on 5 MHz, I'd love for you to help verify the accuracy of this simulator. Feel free to drop by here and share your feedback anytime!

Local Clock Offset ⏱: The accuracy of UTC signals depends on your device's clock. Your device's clock frequency may not be perfectly aligned with the real world, causing drift over time. Networked devices usually use NTP to sync time, but this sync is not real-time, leaving a random offset. You can adjust this offset manually, or click the button to sync with internet time automatically. (Step 1: SYNC your local clock.)
This offset only affects this webpage, not your system clock. Affected by network randomness, a single sync cannot always get a sufficiently accurate offset, and the network error will be displayed on the page. You can try syncing multiple times until the offset remains stable (only single-digit millisecond changes) in a short period with small enough network error, then the offset is accurate enough.

Latency Compensation (ms): This parameter is used to offset the fixed delay of the audio processing. After you have stably synchronized your local clock with internet time, the simulator's sound output may still have a stable lag, which is usually introduced by factors such as browser audio processing and system rendering pipeline. (Step 2: adjust this value until the sound aligns with the real BPM signal.)

This value remains stable as long as your browser and OS audio output settings do not change. If the audio output device or its configuration changes (e.g., switching to Bluetooth headphones, altering sample rate or buffer size), the delay may shift and you will need to recalibrate this compensation. After all three parameters are properly aligned, if you notice a drift later, simply re‑sync your local clock — it usually restores the alignment without touching latency compensation.

DUT1: UT1 correction value. The UTC time we use daily is a uniform time generated by atomic clocks, while UT1 is the astronomical time based on the Earth's rotation. Since the Earth's rotation speed is not uniform, there will be a small difference between the two. To prevent this difference from growing too large, the International Earth Rotation and Reference Systems Service (IERS) adjusts UTC by inserting leap seconds, keeping the difference within 0.9 seconds. The EOP (Earth Orientation Parameters) file records the latest observed value of this difference. With it, we can generate accurate UT1 time service signals, and also embed the correct UT1 correction value in the time code.
You can obtain the latest DUT1 value in two ways:

  • From NTSC: Click the NTSC_EOP_yyyyMMdd.txt link on the adjustment panel – this will take you to the UT1 Measurement and Service Center of NTSC. Look for the latest file that matches the format shown on the page, download it, and then upload it using the file picker. The page will automatically fill the DUT1 value for you.
  • From IERS: Click the IERS/BULLETIN-A link – this is a direct link to the latest IERS Bulletin A (a rapid service/prediction bulletin containing Earth orientation parameters including UT1-UTC). You can either save it directly (right-click → Save as) or open it and save the content as a text file, then upload it using the same file picker. The simulator recognizes both NTSC and IERS formats and will update DUT1 automatically.
(Step 3: for UT1 alignment, download either file and upload it.)
It should be noted that the current simulator cannot implement the leap second announcement function in the time code, nor can it simulate the special signals when a leap second occurs.

设置界面说明

通过调整以下三个参数，你可以让模拟器的输出尽可能贴近真实的BPM信号——包括UTC、时码*、UT1和台站呼号。

*作者注： 说实话，我自己也从未真正听到过真实的BPM时码信号——或者说，它几乎总是被WWV/H的时码副载波淹没了。这个模拟器完全是照着已发表的论文（参考文献[1]）把时码的编码和发播规则“翻译”成声音的。做它的初衷也很简单——就是想亲耳听听BPM的时码到底长什么样，也就是秒脉冲之间那125Hz的低频“嗡嗡”声，顺便了解一下它的工作原理。

不过可以放心，只要你把下面几个参数设置正确，浏览器里播出来的声音和真实的BPM广播就是完全一样的——至少听起来是。如果你真能收听到5MHz上的时码副载波，非常期待你能帮我验证一下这个模拟器的准确性。欢迎来这里随时交流！

本地时钟偏移⏱：UTC信号的准确性取决于你的设备时钟。一般而言，你使用设备的时钟频率与现实世界的频率并不完全一致，这会导致设备时钟长时间运行后与真实时间产生偏移。联网设备一般会使用NTP协议与互联网进行时间同步消除这个偏移，但这个同步动作并不是实时的，这仍然会导致你设备的时钟和真实时间存在随机偏移。你可以通过调整该偏移，或者点击按钮实时地与互联网时间进行同步。（第一步：同步本地时钟。）
该偏移同步只影响网页展示与你收听到的脉冲信号，不会更改你设备上的时钟。受网络连接的随机性影响，单次同步并不总能获取足够精确的偏移量，同步结果会附带显示网络误差。你可以多次尝试同步，直到短时间内多次同步的偏移量保持稳定（仅个位数毫秒的变动），且网络误差足够小，此时的偏移量才足够准确。

延迟补偿(ms)：该参数用于抵消音频链路的固定延迟。在你完成本地时钟与互联网时间的稳定同步后，模拟器的声音输出仍可能存在一个稳定的滞后，这一般由浏览器音频处理、系统渲染链路等因素共同引入。（第二步：调整此值，直到声音与真实BPM信号对齐。）

只要浏览器和操作系统的音频输出设置不变，该值通常不需要频繁调整。如果音频输出设备或其配置发生变化（例如改用蓝牙耳机、更改采样率或缓冲区大小等），延迟可能会改变，需要重新校准。三个参数都调整到位后，如果日后发现信号再次偏移，通常只需重新同步本地时钟即可恢复对齐，无需改动延迟补偿。

DUT1：UT1改正值。我们日常使用的UTC时间是原子钟生成的均匀时间，而UT1是基于地球自转的天文时间，由于地球自转速度并不均匀，两者之间会存在微小的差异。为了避免这个差异持续扩大，国际地球自转服务（IERS）会通过插入闰秒的方式调整UTC，将两者的差值控制在0.9秒以内。EOP（地球定向参数）文件就记录了这个差异的最新观测值，通过它我们可以生成准确的UT1授时信号，同时也能在时码中嵌入正确的UT1改正值。
你可以通过以下两种方式获取最新的DUT1值：

  • 来自国家授时中心（NTSC）： 点击调整面板上的 NTSC_EOP_yyyyMMdd.txt 链接，会跳转到NTSC的UT1测量与服务中心。在页面上找到符合页面显示格式的最新发布文件，下载后使用文件选择器上传，页面会自动填充DUT1值。
  • 来自国际地球自转服务（IERS）： 点击 IERS/BULLETIN-A 链接，这是直达最新 IERS Bulletin A 公报的链接。Bulletin A 是 IERS 发布的快速服务/预测公报，包含地球定向参数（包括 UT1-UTC）的测量值和未来预测。你可以直接保存该文件（右键另存为），或者打开后保存内容为文本文件，然后用同一个文件选择器上传。模拟器能自动识别两种格式，并更新DUT1。
（第三步：如需对齐UT1信号，下载任一文件并上传。）
需要说明的是，当前模拟器无法实现时码中的闰秒预告功能，也无法模拟闰秒发生时的特殊信号。
